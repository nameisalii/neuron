Chapter 01

Where does Neuron live?

Neuron is not a terminal tool, not a browser extension, and not a standalone chat app. It lives in four places simultaneously — and most people never leave the tools they already use.

💬
Primary surface · Most used

The Slack bot

Most people interact with Neuron entirely through Slack. Type /neuron what is our refund policy in any channel or DM — and get a verified answer in 2 seconds. Neuron also sends proactive messages: task reminders, bug owner alerts, offboarding warnings.

Best for: Every day, every team member
🖥️
Secondary surface · Power users

The web dashboard

A clean web app at app.neuron.ai. This is where admins manage integrations, review the full knowledge graph, resolve conflicts, verify rules, see analytics, and manage team settings. Think of it as the "control room."

Best for: Admins, team leads, onboarding setup
⚡
Developer surface · For AI agents

The API

A REST + MCP API that AI agents, internal tools, and automations query directly. Your Stripe webhook, your support bot, your custom agent — they all ask Neuron for context before acting. Returns structured JSON with confidence score, source, and owner.

Best for: Developers building on top of Neuron
🔔
Passive surface · Always on

Background watcher

Neuron runs silently in the background 24/7 — watching connected channels, detecting bugs, extracting decisions, flagging conflicts. No interface needed. You only see it when it has something important to tell you. Like a radar, not a dashboard.

Best for: Nobody has to do anything — it just works

The simple mental model: Neuron is like a very smart new team member who reads everything, remembers everything, never gets tired, and answers any question about how your company works — in Slack, in a web app, or via API. You don't install it on your laptop. You don't use a new app. You just ask it things where you already work.

Chapter 02

How a company signs up and sets up

From zero to fully running in under 30 minutes. No engineers needed for setup. No data migration. No manual entry.

1

Sign up at app.neuron.ai

The admin (usually a team lead, CTO, or head of support) goes to the Neuron website, clicks "Get Started," and signs in with their company Google or Microsoft account. Takes 60 seconds. No credit card needed to start.

What happens: A company workspace is created. The admin becomes the first user. Everything else is empty — Neuron hasn't read anything yet.
2

Connect your tools in one click each

The setup screen shows big "Connect" buttons for each integration: Slack, Notion, GitHub, Google Drive, Jira. The admin clicks each one and approves the OAuth permissions. That's it. No API keys, no manual configuration, no code.

For Slack specifically: Clicking "Connect Slack" opens Slack's standard permissions screen — same as adding any other Slack app. Admin approves, and Neuron's bot joins the workspace instantly. It starts listening silently from that moment on.
3

Choose which channels and spaces to monitor

The admin selects which Slack channels, Notion spaces, and GitHub repos Neuron should read. They can start narrow — just #support and the support Notion space — and expand later. Private channels require an explicit opt-in from that channel's owner.

Privacy controls: Neuron only reads what it's explicitly given access to. No personal DMs unless the user specifically asks Neuron a question there. Companies set their own boundaries.
4

Neuron reads everything and builds the first knowledge graph

Neuron processes the last 90 days of activity across all connected tools. It extracts rules, decisions, ownership, and processes — and builds the first version of the company's knowledge graph. This takes 15–45 minutes depending on company size.

What the admin sees: A progress screen showing "Processing Slack (12,430 messages)... Extracting rules... Found 47 knowledge items... 3 conflicts detected." When done: a dashboard showing the knowledge graph with a summary.
5

Verify the first batch of knowledge items

Neuron presents the top 10–20 most important extracted knowledge items and asks owners to verify them with one click. This first verification pass takes about 10 minutes and immediately makes Neuron reliable for the whole team.

Slack notification sent to each owner: "Hey — I found this rule in your area. Is this still accurate? [✓ Yes, correct] [✏️ Edit] [✗ Outdated]" — one tap, done.
6

Invite the team — they're already set up

The admin invites team members via email or Slack. Each person gets a DM from Neuron: "Hi, I'm Neuron — your company's knowledge brain. Ask me anything with /neuron or just DM me." No setup required for regular users. They just start using it.

From this point: Neuron runs in the background forever. Every new message, doc update, and PR is processed automatically. The knowledge graph improves daily with zero effort from the team.

Chapter 04

How it's built under the hood

Simple enough for two engineers to build in 8 weeks. Scalable enough to run for thousands of companies.

📥
Connectors
Slack, Notion, GitHub, Drive, Jira OAuth integrations read data in real time
→
🧠
Extraction engine
LLM pipeline reads raw content, extracts structured rules + owners + conditions
→
🗃️
Knowledge graph
PostgreSQL stores structured rules. Pinecone stores embeddings for semantic search
→
⚡
Delivery layer
Slack bot, web dashboard, REST + MCP API — all serve from the same knowledge graph
Frontend Next.js + Tailwind CSS The web dashboard at app.neuron.ai
Backend Node.js (API) + Python (extraction pipeline) API handles requests; Python runs the LLM extraction jobs
LLM Claude API (Sonnet 4) Extracts structured rules from raw text. Best at following complex schemas.
Structured DB PostgreSQL Stores the knowledge graph: rules, owners, sources, verification status, timestamps
Vector DB Pinecone Embeddings for semantic search — "find rules about refunds" even if word not used
Auth Clerk + OAuth2 Company SSO, Google/Microsoft login, per-user permissions
Queues Redis + BullMQ Background job queue for processing large ingestion batches
Hosting Vercel (frontend) + Railway (backend) Zero-config deployment. Scales automatically. Cheap to start.
Slack bot Slack Bolt SDK Handles slash commands, DMs, event subscriptions from Slack
Monitoring Sentry + PostHog Error tracking + product analytics
